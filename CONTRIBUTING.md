# Contributing to OpenInfra Logger

First off, thank you for considering contributing to OpenInfra Logger! It's people like you that make open-source a great community.

## Code of Conduct

By participating in this project, you are expected to uphold our [Code of Conduct](CODE_OF_CONDUCT.md).

## How to Contribute

### Reporting Bugs
- Ensure the bug was not already reported by searching on GitHub under Issues.
- Open a new Issue with a clear title and description. Include a code snippet demonstrating the bug.

### Suggesting Enhancements
- Open a new Issue and describe the enhancement. Explain *why* this enhancement would be useful.

### Pull Requests
1. Fork the repo and create your branch from `main`.
2. Ensure your code adheres to the existing coding standards.
3. Write clear, descriptive commit messages.
4. If you've added code that should be tested, add tests.
5. Update the documentation if necessary.
6. Issue a pull request.

## Release Policy
We follow [Semantic Versioning](https://semver.org/). Releases are automated via CI/CD pipelines upon merging into the `main` branch.
